# Deliver first-party Design Packs as single Design Contracts

AgentKogei will deliver each first-party Pack Release from the Official Catalog as raw Markdown and install it as one root `DESIGN.md`. Each contract directly targets the MVP React or Next.js, Tailwind CSS v4, and shadcn/ui stack; there are no manifests, Stack Adapters, third-party Pack Sources, executable hooks, separate resources, or `.agentkogei` state. The CLI may maintain one marked `AGENTS.md` reference for discovery, but the Design Contract carries only human-readable provenance and remains inert.

`add` is the only Design Pack lifecycle command. It resolves only Official Catalog identities, previews and confirms creation or replacement in the current directory, requires `--yes --force` for non-interactive replacement, and can select either the latest release or an explicit version. Removal is the manual deletion of `DESIGN.md` and its marked `AGENTS.md` reference. A Project License arises when an entitled Builder installs a Premium Design Pack, but neither the CLI nor AgentKogei creates or retains a Project identifier; AgentKogei may record the Builder, Pack Release, and Installation event for entitlement and audit purposes. The CLI never sends Project names, paths, Git remotes, file contents, prompts, generated interfaces, or dependency lists, and diagnostics remain opt-in. We accept the loss of managed status, updates, Detachment, integrity state, binary assets, third-party distribution, and framework-neutral adapters because one portable artifact and one operation make the product materially easier to understand and adopt.
